## Структура репозитория `VertexAI`

**Цель**: единая и понятная структура для команды (бэкенд, фронтенд, данные, документация и инфраструктура) в духе плана MVP.

```text
vertex/
│
├── 📂 src/                          # Исходный код бэкенда
│   ├── 📂 core/                     # Ядро бизнес-логики
│   │   ├── agent.py                 # Клиент для LLM API (Qwen)
│   │   ├── extractor.py             # Извлечение текста из PDF
│   │   ├── ratios.py                # Расчёт финансовых коэффициентов
│   │   ├── scoring.py               # Интегральный скоринг компании
│   │   ├── nlp.py                   # NLP-анализ рисков и возможностей
│   │   ├── recommendations.py       # Генерация рекомендаций
│   │   └── prompts.py               # Промпты для LLM
│   │
│   ├── 📂 models/                   # Модели данных
│   │   ├── schemas.py               # Pydantic схемы для API
│   │   ├── requests.py              # Модели запросов
│   │   └── settings.py              # Настройки приложения (pydantic-settings)
│   │
│   ├── 📂 controllers/              # Контроллеры
│   │   └── analyze.py               # Логика анализа PDF
│   │
│   ├── 📂 routers/                  # API роутеры
│   │   ├── analyze.py               # Endpoints для анализа
│   │   └── system.py                # Системные endpoints (health)
│   │
│   ├── 📂 exceptions/               # Пользовательские исключения
│   │   └── PdfExtractException.py   # Исключение при ошибке извлечения PDF
│   │
│   ├── 📂 database/                 # Модели базы данных (опционально)
│   │   ├── project.py               # Модель проекта
│   │   └── user.py                  # Модель пользователя
│   │
│   └── app.py                       # Точка входа FastAPI приложения
│
├── 📂 frontend/                     # Фронтенд приложения
│   ├── index.html                   # Главная страница
│   ├── style.css                    # Стили (Vertex дизайн-система)
│   ├── script.js                    # Клиентская логика (fetch API)
│   └── nginx.conf                   # Конфигурация nginx для Docker
│
├── 📂 data/                         # Данные
│   └── pdf_samples/                 # Тестовые PDF-отчёты
│
│
├── 🔧 .env                          # Переменные окружения (НЕ в git)
├── 📋 .env.example                  # Шаблон переменных окружения
├── 🚫 .gitignore                    # Игнорируемые файлы git
│
├──  requirements.txt              # Python зависимости
├── 🐳 Dockerfile                    # Docker образ бэкенда
├── 🐳 docker-compose.yml            # Docker Compose конфигурация
│
├── 📖 README.md                     # Основная документация
├──  REPO_STRUCTURE.md             # Детальная структура репозитория
└── 📖 LICENSE                       # Лицензия проекта
```

### Основные принципы

- **Ясное разделение зон ответственности**:
  - `backend/` — весь серверный код (FastAPI, модули анализа, схемы).
  - `frontend/` — статический фронтенд без фреймворков (на этапе MVP).
  - `data/` — примеры входных данных и их описание (для аналитиков и тестов).
  - `infra/` — Docker и прочая инфраструктура .

- **Готовность к росту**:
  - При появлении БД или отдельных сервисов можно добавлять подпапки в `backend/` и `infra/docker/` без пересмотра общей структуры.
