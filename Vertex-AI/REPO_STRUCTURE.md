## Структура репозитория `neo-fin-ai`

**Цель**: единая и понятная структура для команды (бэкенд, фронтенд, данные, документация и инфраструктура) в духе плана MVP.

```text
neo-fin-ai/
  backend/
    app/
      api/
        __init__.py
        endpoints.py          # REST-эндпоинты FastAPI (в т.ч. /analyze)
      core/
        __init__.py
        ocr.py                # OCR и работа с Tesseract
        extractor.py          # извлечение показателей из текста
        ratios.py             # расчёт финансовых коэффициентов
        scoring.py            # интегральный скоринг
        recommendations.py    # формирование рекомендаций
        nlp.py                # работа с локальной LLM (Ollama)
        news.py               # (опц.) модуль новостей и сентимента
      models/
        __init__.py
        schemas.py            # Pydantic-схемы запросов/ответов API
      main.py                 # создание FastAPI-приложения, точки входа
    requirements.txt          # зависимости бэкенда (FastAPI, pdfplumber, pytesseract, и т.д.)

  frontend/
    index.html                # базовая HTML-страница
    style.css                 # минимальная стилизация
    script.js                 # логика загрузки PDF и работы с API

  data/
    pdf_samples/              # 5–7 примерных отчётов PDF (сканы и текстовые)
    README.md                 # описание структуры данных и правил именования

  docs/
    PLAN.md                   # план (копия/ссылка на основной документ, не правится кодом)
    WEEK1_TASKS_DETAILED.md   # детализированные задачи недели 1
    ANALYTICS_SPECS.md        # показатели, коэффициенты, скоринг, требования к NLP (по мере появления)

  infra/
    docker/
      backend.Dockerfile      # Dockerfile для бэкенда (неделя 3)
      docker-compose.yml      # docker-compose (backend, frontend, ollama и т.д.)

  .gitignore                  # игнор временных файлов, виртуальных окружений и т.п.
  README.md                   # корневое описание проекта, запуск, архитектура
```

### Основные принципы

- **Ясное разделение зон ответственности**:
  - `backend/` — весь серверный код (FastAPI, модули анализа, схемы).
  - `frontend/` — статический фронтенд без фреймворков (на этапе MVP).
  - `data/` — примеры входных данных и их описание (для аналитиков и тестов).
  - `docs/` — живая документация (планы, спецификации, сценарии).
  - `infra/` — Docker и прочая инфраструктура (подключается с 3-й недели).

- **Совместимость с планом по неделям**:
  - Неделя 1 создаёт каркас: `backend/app/*`, базовый фронтенд, первые документы в `docs/` и `data/`.
  - Недели 2–3 наращивают функциональность внутри уже определённой структуры (без изменения корневой схемы каталогов).

- **Готовность к росту**:
  - При появлении БД или отдельных сервисов можно добавлять подпапки в `backend/` и `infra/docker/` без пересмотра общей структуры.
